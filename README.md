# 📟 Ridi Paper E-Ink Server Dashboard

이 프로젝트는 구형 전자책 단말기인 **리디북스 페이퍼(1세대)**를 재활용하여 서버 상태를 실시간으로 모니터링하는 **상시거치형 대시보드**로 만드는 프로젝트입니다. 안드로이드 4.2.2의 한계와 전자잉크 디스플레이의 특성을 고려하여 최적화되었습니다.

---

## 🏗 작동 원리 (Architecture)

본 시스템은 서버-클라이언트 모델을 따르며, 구형 기기의 성능을 고려하여 모든 복잡한 처리는 서버에서 수행합니다.

### 1. Server-side (`stats.php`)
*   **역할**: 서버의 시스템 자원(CPU, Memory, Disk, Uptime)을 수집하여 JSON 형태로 제공합니다.
*   **로직**: 리눅스 시스템 명령어를 PHP에서 실행하고 파싱하여 클라이언트에 최적화된 데이터를 응답합니다.

### 2. Client-side (`eink.html`)
*   **역할**: 서버에서 받은 데이터를 시각화하여 전자잉크 디스플레이에 출력합니다.
*   **레거시 지원**: 최신 브라우저 기술(Grid, Flexbox, Fetch API)을 지원하지 않는 환경에 맞춰 **Float 레이아웃**과 **XMLHttpRequest(XHR)**를 사용합니다.
*   **E-Ink 최적화**: 
    *   **고대비 UI**: 16-단계 그레이스케일을 고려한 흑백 위주의 디자인.
    *   **안티 고스트**: 주기적인 전체 페이지 새로고침(`meta refresh`)으로 전자잉크 특유의 잔상을 제거합니다.
    *   **가독성**: 1024x758 해상도(300 PPI)에서 멀리서도 잘 보이도록 대형 타이포그래피를 적용했습니다.

### 3. Browser Launcher Principle (`Go_Dashboard.pdf` / `VibeDashboard.epub`)
*   **원리 (The 'Exploit')**: 리디북스 페이퍼는 일반적인 방법으로는 인터넷 브라우저에 접속할 수 없습니다. 하지만 단말기에 내장된 **PDF/EPUB 뷰어는 파일 내부의 하이퍼링크(Hyperlink)를 처리하는 기능**이 있습니다.
*   **메커니즘**:
    1. 하이퍼링크가 포함된 PDF 문서를 생성합니다.
    2. 단말기 내장 뷰어로 해당 PDF를 열면, 뷰어 엔진이 링크를 인식합니다.
    3. 사용자가 링크를 터치하면, 안드로이드 시스템은 해당 URL을 처리할 수 있는 기본 앱(임베디드 웹브라우저)을 호출(Intent)합니다.
    4. 결과적으로 시스템의 숨겨진 웹 브라우저가 실행되면서 외부 웹서버인 대시보드에 접속하게 됩니다.
*   **파일 특징**: `Go_Dashboard.pdf`는 시각적인 디자인보다는 **정확한 `http://` 링크를 포함하는 것**이 핵심입니다. (HTTPS 호환성 이슈를 고려해 `http` 링크를 권장합니다.)

## 🚀 사용 도구 및 기술 스택

- **Hardware**: Ridibooks Paper 1st Gen (Android 4.2.2 Jelly Bean)
- **Frontend**: HTML5, Vanilla CSS (Float-based), Legacy JavaScript (ES5, XHR)
- **Backend**: PHP, Linux System Commands
- **Transfer**: OpenMTP (Android File Transfer 대체용 오픈소스 도구)

## 🛠 설치 및 사용 방법

### 1. 서버 설정
*   `stats.php`와 `eink.html`을 모니터링 대상 서버의 웹 루트(예: `/var/www/html/`)에 업로드합니다.
*   웹 브라우저에서 `http://your-server-ip/eink.html`이 정상적으로 수치를 가져오는지 확인합니다.

### 2. 런처 생성
*   `eink.html`의 URL을 하이퍼링크로 포함하는 PDF 또는 EPUB 파일을 생성합니다.
*   **프리미엄 PDF 생성**: `Launcher_Template.html`을 활용하면 더 고품질의 런처를 만들 수 있습니다.
    1.  `Launcher_Template.html`을 브라우저(크롬 등)로 엽니다.
    2.  인쇄(Cmd + P) 메뉴에서 'PDF로 저장'을 선택합니다.
    3.  `Go_Dashboard.pdf`라는 이름으로 저장하여 기기에 전송합니다.

### 3. 기기 전송 및 실행
1.  리디북스 단말기를 PC에 연결하고 **OpenMTP** 등의 도구를 사용하여 생성한 런처 파일을 단말기 내 `Documents` 폴더로 복사합니다.
2.  단말기 설정에서 **Wi-Fi에 연결**합니다.
3.  단말기의 독서 목록에서 전송한 파일을 열고, 포함된 링크를 터치합니다.
4.  시스템 브라우저가 실행되면 대시보드가 나타납니다.

## 📋 주요 기능
- **실시간 데이터**: 1분마다 서버 자원 자동 갱신.
- **디지털 시계**: 1024x758 최적화 대형 시계 및 날짜 위젯.
- **상태 시각화**: 5자리 정밀 CPU 로드 및 그레이스케일 프로그레스 바.
- **친절한 UI**: 서버 상태(Optimal)를 직관적인 스마일 아이콘과 함께 표시.
