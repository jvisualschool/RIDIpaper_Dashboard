# 📖 리디북스 페이퍼(2016) 서버 대시보드 구축 가이드

2016년식 리디북스 페이퍼 1세대를 활용하여 AWS Lightsail 리눅스 서버의 상태를 상시 모니터링하는 전용 대시보드를 구축하는 과정입니다.

## 1. 개요

- **장치**: 리디북스 페이퍼 1세대 (안드로이드 4.2.2 젤리빈 기반)
- **목표**: 서버의 `stats.php` 데이터를 가져와서 전자잉크(E-ink)에 최적화된 화면으로 출력
- **방식**: 순정 상태의 리더기에서 EPUB 링크를 통해 내장 브라우저를 호출하여 리다이렉트

## 2. 서버 측 설정 (Lightsail)

전자잉크 디스플레이의 특성(낮은 주사율, 그레이스케일)을 고려하여 고대비 흑백 페이지를 생성합니다.

- **파일 경로**: `/home/bitnami/htdocs/eink.html`
- **주요 설정**:
  - `1분 자동 새로고침` (`meta http-equiv="refresh" content="60"`)
  - `고대비 디자인` (백색 배경, 흑색 텍스트)
  - `데이터 연동`: JavaScript `fetch`를 통해 `stats.php` 호출

## 3. 리더기용 '입장권' EPUB 생성

리더기의 [내서재]에서 열어 브라우저를 실행시킬 용도의 EPUB 파일입니다.

- **파일명**: `VibeDashboard.epub`
- **동작**: 책을 열어 중앙의 링크를 클릭하면 브라우저가 실행되어 `https://jvibeschool.net/eink.html`로 접속

## 4. 맥북(macOS)에서 데이터 전송 준비 (ADB)

맥북은 안드로이드 기기를 일반 드라이브로 인식하지 못하므로 ADB(Android Debug Bridge)를 활용합니다.

### 4.1 ADB 설치

```bash
brew install --cask android-platform-tools
```

### 4.2 리더기 설정 (개발자 모드 해제)

1.  리더기 **[설정]** ➡️ **[기기 정보]** 진입
2.  **'빌드 번호'** (또는 소프트웨어 정보 내 빌드 번호) 항목을 **7번 연속 터치**
3.  "개발자가 되었습니다" 메시지 확인
4.  **[개발자 옵션]** ➡️ **[USB 디버깅]** 활성으로 변경

## 5. 파일 전송 및 실행

리더기가 인식되면 터미널에서 다음 명령어를 실행하여 파일을 전송합니다.

```bash
# 기기 연결 확인
adb devices

# 파일 전송
adb push ./VibeDashboard.epub /sdcard/Documents/
```

## 6. 최종 실행

1.  리더기 **[내서재]** ➡️ **[내 문서 추가]** (또는 새로고침)
2.  `VibeDashboard` 책을 연 후 **[대시보드 입장]** 터치
3.  브라우저가 열리면 상시 전원을 연결하여 관제 모니터로 활용
